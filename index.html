课堂边界说明：CRC-4 生成多项式为 x^4 + x + 1（二进制 10011），CRC 主要用于检测错误，不修复数据；校验失败时原错误数据仍在；不宣称能检测所有可能的错误组合。FEC（三次重复码 / 汉明码）利用冗余修复一定能力范围内的错误——重复码 2 或 3 副本同时错会恢复失败；汉明(7,4) 每组最多保证纠 1 位错，≥2 位错时超出单错纠正能力，可能留下错误或发生误纠正（不是"必然以某种固定方式失败"）。增加保护会带来传输开销（发送总 bit 数变多）。

固定映射

BPSK：左边的点 = 0，右边的点 = 1。

QPSK（Gray）：右上 = 00，左上 = 01，左下 = 11，右下 = 10。

正确公式

信道 BER = 信道中被翻转的编码 bit 数 ÷ 编码后发送总 bit 数。

Payload BER = 恢复后仍错的原始数据 bit 数 ÷ 原始 payload bit 数。

CRC-4（x⁴+x+1，二进制 10011）：只检测错误，不自动修复，不宣称能检测所有可能的错误组合。

FEC：利用冗余修复一定能力范围内的错误，不是任意数量都能修复。

四关提示（一级 / 二级 / 三级）

①符号配对赛：先数这种调制一共有几个理想点 → QPSK 四个状态需要两个bit → 看右上左上左下右下固定Gray映射。

②点云判决挑战：先看这个点落在哪个判决区域 → 接收端判为距离最近的理想点 → 不要猜发送端原本想发什么。

③BER侦探：从左到右逐位比较 → 先数出不同位置再看总位数 → BER分母是总bit数不是错误数或正确数。

④海岛通信指挥官：先判断哪种数据最不能出错 → 再看链路和电量是否支持高开销方案 → 先保位置和文字再考虑语音图片。
