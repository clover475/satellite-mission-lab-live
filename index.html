课堂边界说明：CRC-4 生成多项式为 x^4 + x + 1（二进制 10011），CRC 主要用于检测错误，不修复数据；校验失败时原错误数据仍在；不宣称能检测所有可能的错误组合。FEC（三次重复码 / 汉明码）利用冗余修复一定能力范围内的错误——重复码 2 或 3 副本同时错会恢复失败；汉明(7,4) 每组最多保证纠 1 位错，≥2 位错时超出单错纠正能力，可能留下错误或发生误纠正（不是"必然以某种固定方式失败"）。增加保护会带来传输开销（发送总 bit 数变多）。

固定映射

BPSK：左边的点 = 0，右边的点 = 1。

QPSK（Gray）：右上 = 00，左上 = 01，左下 = 11，右下 = 10。

正确公式

信道 BER = 信道中被翻转的编码 bit 数 ÷ 编码后发送总 bit 数。

Payload BER = 恢复后仍错的原始数据 bit 数 ÷ 原始 payload bit 数。

CRC-4（x⁴+x+1，二进制 10011）：只检测错误，不自动修复，不宣称能检测所有可能的错误组合。

FEC：利用冗余修复一定能力范围内的错误，不是任意数量都能修复。在编码设计合理的情况下，增加冗余通常可以为更强的检错或纠错能力创造条件，但能力不仅取决于冗余数量，也取决于编码规则的设计。

四关提示（一级 / 二级 / 三级）

①符号配对赛：先数这种调制一共有几个理想点 → QPSK 四个状态需要两个bit → 看右上左上左下右下固定Gray映射。

③BER侦探：从左到右逐位比较 → 先数出不同位置再看总位数 → BER分母是总bit数不是错误数或正确数。

    第④课 · 芯片架构调度台 · 快速答案卡（打印版）

    统一口径

    CPU/DSP/FPGA/专用硬件加速器统一称"四类计算资源或实现选择"，不叫"四种平行芯片"。CPU/DSP/加速器可能集成在同一 SoC 内；FPGA 可能是外部器件，也有内置 FPGA 的 SoC，但并非所有 SoC 都带 FPGA。

    七维指标

    延迟↓ 吞吐量↑ 功耗↓ 灵活性↑ 前期开发成本↓ 量产单台成本↓ 数据搬运压力（低/中/高）。全部为课堂规则模型中的相对等级，不是真实瓦数/毫秒/报价。

    三场景权重

    科研原型：重灵活性与前期开发成本。应急救援：重延迟、吞吐量、功耗与可靠运行。十万台量产：重功耗与单台成本，容忍更高前期开发投入。

    典型任务口径

    协议/界面/连接管理→CPU；滤波/FFT→DSP 或 FPGA，专用硬件也可以；解调/纠错→科研原型 DSP/FPGA，数据量大实时性高 FPGA/专用硬件，稳定量产专用硬件更优；波束方向策略→CPU/DSP，多通道高速更新→FPGA/专用硬件；温度监控→CPU（真实产品可能有独立管理控制器）；数据搬运管理→CPU 负责配置，真实高速搬运由 DMA/内存控制器完成。
