## 二、实验模式讲师指引

### 实验① 星座图与判决

- **实验目的**：让学生看到"噪声让接收点偏离理想位置，点跨过判决边界才会被判错"，不是"噪声直接把 bit 改错"。
- **建议参数**：先固定种子演示 σ=0（理想点集中）→ 中噪声预设 → 高噪声预设，BPSK 演示完再切 QPSK 对比。
- **操作步骤**：① 展示 σ=0 理想点；② 逐步提高噪声观察点云扩散；③ 比较 BPSK 与 QPSK 同噪声下的误判情况；④ 让学生指出哪些点越过了判决边界；⑤ 需要"先猜后核对"时，点"隐藏答案"再点"显示答案"核对。
- **正常现象**：噪声越大，红点（判决错误）越多；QPSK 比 BPSK 更容易在同噪声下出错（相邻理想点间距更小）。
- **正确结论**：噪声会让接收点偏离理想位置。只要点仍留在原来的判决区域，接收端通常还能判断正确；当点越过边界，就可能被判断成另一个符号。
- **学生可能误解**：以为"噪声直接改变了 bit 的值"，而不是"噪声先让接收点移动位置，判决错误后才产生 bit 错误"。
- **20 秒逐字稿**：这个星座图上，蓝色点是发送端理想发出的符号。噪声会把接收点往四周推。只要红色的接收点还留在原来的区域里，接收端还是能判断正确；一旦跨过虚线判决边界，才会被判成另一个符号。
- **60 秒逐字稿**：我们发送的每个符号，理论上应该落在星座图上的一个固定理想点。但信道里的噪声会让接收点"抖动"，偏离理想位置，抖动幅度由这个 σ 滑杆控制。判决规则很简单：接收端只看这个抖动后的点距离哪个理想点最近，就判成哪个符号。只要抖动没有让点跨过虚线（判决边界），结果仍然正确；一旦跨过边界，就会被判成邻近的另一个符号，这时才会产生 bit 错误。注意 QPSK 因为理想点之间挨得更近，同样的噪声下更容易被跨界判错，这不代表 QPSK "更差"，只是阶数更高、抗噪能力的代价。
- **技术边界**：本实验的噪声 σ 只影响这个星座图的点云和判决，和实验②③里的"bit 翻转概率"是两个完全独立的变量，互不影响。

### 实验② 误码与 BER

- **实验目的**：让学生亲手数出"错了几位、共传了几位"，理解 BER 只是这两个数字的比值。
- **建议参数**：先用「A 简单 bit 串」层级（发送 10110010），手动翻转 2 位（例如变成 11110000），全班一起手算 BER；再切换到「B 紧急消息 TEAM SAFE」层级演示 45 bit 场景。
- **操作步骤**：① 选难度层级（简单 bit 串 / TEAM SAFE）；② A 手动翻转模式：点击 bit 制造错误，点"隐藏错误标记"让学生自己先找错误位置，再点"显示错误位置"核对，点"计算 BER"前不要提前公布；③ B 随机信道模式：调节独立翻转概率 p，可固定随机种子复现；④ 强调分子（信道中被翻转的 bit 数）和分母（编码后发送总 bit 数）分别是什么。
- **正常现象**：p 越大，被标红的翻转位越多，信道 BER 越高；本实验不加保护，所以"信道 BER"和"Payload BER"数值相同。
- **正确结论**：BER 只回答一件事：总共传了多少 bit，其中有多少 bit 接错了。
- **学生可能误解**：把"信道 BER"和"Payload BER"混为一谈——本实验因为没有加保护方案，两者数值相同，但下一个实验（CRC/FEC）加了保护之后，两者会出现差异，这是重要的对比点。
- **20 秒逐字稿**：这里发送了一串 bit。信道翻转了其中几位（红色标出），这就是"信道 BER"：翻转数除以总位数。因为这次没有加任何保护，最后恢复出来的数据错误位数和信道翻转位数完全一样。
- **60 秒逐字稿**：误码率（BER）的计算只需要两个数字：分子是错误的 bit 数，分母是传输的总 bit 数。本实验里，"信道 BER"指的是信道中被翻转的编码 bit 数除以编码后发送的总 bit 数；"Payload BER"指的是恢复之后仍然错误的原始数据 bit 数除以原始 payload 的 bit 数。因为这个实验不加任何保护方案，信道翻转多少位，恢复出来就错多少位，所以这两个数字现在相等——但这不是巧合，而是"没有保护"这个前提决定的。等到下一个实验加了 CRC 或三次重复码之后，你们会看到这两个数字开始不一样。
- **技术边界**：本实验的 bit 翻转概率 p（或手动翻转）与实验①的连续噪声 σ 是两套完全独立的模型，bit 翻转是课堂简化模型，不代表真实无线信道的全部物理过程。

### 实验③ CRC 与 FEC

- **实验目的**：让学生分清"检错"（CRC）和"纠错"（三次重复码/汉明码）的区别，以及"多保护 = 多开销"的取舍。
- **建议参数**：先演示 A 不加保护（对照基线）→ B 教学版 CRC-4（生成多项式 x⁴+x+1，二进制 10011）→ C 三次重复码 → 可选展开 D 汉明(7,4)；p 建议先设 10%-20% 观察效果，再跑批量统计 100/500 次看趋势。
- **操作步骤**：① 选保护方案；② 调节 bit 翻转概率 p；③ 先跑单次案例，观察发送/信道/恢复三行 bit 和恢复文本；④ 再跑批量统计 100 或 500 次，比较完整恢复率、平均 Payload BER、发送总 bit 数、检错/纠错成功率；⑤ 强调"CRC 只报告校验失败，不修复数据"。
- **正常现象**：CRC 校验失败时，原始错误数据原样保留，没有被"修好"；三次重复码大部分情况能修复单个副本错误，但如果同一 bit 的 3 个副本里有 2 个或以上被翻转，多数表决会得到错误结果；汉明(7,4)（可选扩展）单个 codeword 内 1 位错必定纠对，2 位及以上错误超出单错纠正能力，可能留下错误或发生误纠正。
- **正确结论**：CRC 主要用于检测错误，不修复数据；FEC（三次重复码/汉明码）利用冗余修复一定能力范围内的错误；增加保护会带来传输开销（发送总 bit 数变多）。
- **学生可能误解**：以为"CRC 校验失败=自动把数据修好了"；以为"重复码/汉明码可以修复任意数量的错误"；以为"CRC-4 能检测所有可能的错误组合"（CRC-4 只保证检出单 bit 错误和大多数、并非全部的多 bit 错误组合）。
- **20 秒逐字稿**：CRC 只负责"体检"，告诉你数据可能坏了，它自己不会把坏的数据修好。三次重复码是真正会"修"的方法，把每个 bit 发三遍，多数表决——但如果三份里坏了两份以上，修出来的结果也会是错的。
- **60 秒逐字稿**：这个实验对比三种保护方式。A 不加保护，发送量最小，但没有任何检错或纠错能力。B 是教学版 CRC-4，生成多项式是 x⁴+x+1（二进制 10011），接收端会用同样的多项式重新计算一次余数，和收到的校验位比较：一致就"通过"，不一致就报告"校验失败"——但无论通过还是失败，CRC 都不会去改动任何数据 bit，失败时原始错误数据原样还在。C 是三次重复码，把每个 bit 重复发送三次，接收端用多数表决决定这一位是 0 还是 1，如果三份里只错了一份，多数表决能把它修回来；但如果错了两份或三份，修出来的反而是错的。批量统计模式能让大家看到"单次结果有随机性，多次统计才能看出总体趋势和平均可靠性"这句话的真正含义。
- **技术边界**：CRC-4 只检错不纠错，不宣称能检测所有可能的错误组合；三次重复码/汉明码的纠错能力都有明确上限，超出上限可能留下错误或发生误纠正，不是"冗余越多一定越好"，而是要在可靠性和开销之间取舍。

### 第②关 点云判决挑战（8 题，判断题）

- **规则**：给一个带噪声的接收点（发送端实际发送的符号暂时隐藏），学生判断"接收端会判成哪个 symbol"（不问"发送端原来想发什么"）。题目不生成恰好落在判决边界上的歧义点。
- **固定映射表**：与第①关相同。
- **评分规则**：同第①关（2 / 1 / 0 / 答错不扣）。
- **三级提示**（本关固定算法）：一级——先看这个点落在哪个判决区域；二级——接收端通常判为距离最近的理想点；三级——不要猜发送端原本想发什么，只判断接收点现在落在哪一侧、离哪个理想点更近。
- **8 题逐题数据**：
  1. [难度简单] 发送符号「0」，接收点判决为「1」，产生符号错误（1 个 bit 错误）
     - 20 秒：噪声把接收点推离了理想位置。接收端只看这个点现在离哪个理想点最近，本题判为「1」。
  2. [难度中等] 发送符号「0」，接收点判决为「1」，产生符号错误（1 个 bit 错误）
     - 20 秒：噪声把接收点推离了理想位置。接收端只看这个点现在离哪个理想点最近，本题判为「1」。
  3. [难度困难] 发送符号「0」，接收点判决为「0」，判决正确（0 个 bit 错误）
     - 20 秒：噪声把接收点推离了理想位置。接收端只看这个点现在离哪个理想点最近，本题判为「0」。
  4. [难度简单] 发送符号「00」，接收点判决为「00」，判决正确（0 个 bit 错误）
     - 20 秒：噪声把接收点推离了理想位置。接收端只看这个点现在离哪个理想点最近，本题判为「00」。
  5. [难度中等] 发送符号「00」，接收点判决为「00」，判决正确（0 个 bit 错误）

### 第③关 BER 侦探（8 题，判断题）

- **规则**：给发送串和接收串，学生要：① 点击所有错误位置 ② 填错误 bit 数 ③ 选出正确的 BER。三项都对才算全部正确。
- **正确答案逻辑**：逐位比较发送串和接收串找出所有不同的位置；BER = 错误 bit 数 ÷ 传输总 bit 数。全部 8 题只使用简单分数：**1/8、2/8、3/12、2/16、4/16**（2026-07-20 规格固定，同一比例可用不同分母表示：2/8=3/12=4/16=25%；1/8=2/16=12.5%），不出现循环小数。
- **评分规则**：同第①关（三项全对才算"答对"，2 / 1 / 0 / 答错不扣）。
- **三级提示**（本关固定算法）：一级——从左到右逐位比较；二级——先数出不同的位置，再看总共多少位；三级——BER 的分母不是错误数也不是正确数，而是总 bit 数。
- **8 题逐题数据**：
  1. 发送 10101010｜接收 10101110｜错误位置(从0计数) 5｜BER = 1/8
  2. 发送 11001100｜接收 10001110｜错误位置(从0计数) 1、6｜BER = 2/8
  3. 发送 100110110010｜接收 101110100000｜错误位置(从0计数) 2、7、10｜BER = 3/12
  4. 发送 1011001011010011｜接收 1010001011011011｜错误位置(从0计数) 3、12｜BER = 2/16
  5. 发送 01101001｜接收 01001001｜错误位置(从0计数) 2｜BER = 1/8
  6. 发送 00110110｜接收 10110010｜错误位置(从0计数) 0、5｜BER = 2/8
  7. 发送 011001011001｜接收 001011011101｜错误位置(从0计数) 1、4、9｜BER = 3/12
  8. 发送 1100101100110100｜接收 1110100101110000｜错误位置(从0计数) 2、6、9、13｜BER = 4/16
- **复盘固定句**：BER 描述总 bit 中有多少比例接错。它不是网速，也不直接等于信号强度。
- 若题目涉及编码保护（CRC/FEC），结果需同时显示信道 BER 与 Payload BER——本关 8 题为原始信道场景不涉及编码，两个 BER 的对比留给实验③演示（详见第二节实验②③）。

### 第④关 海岛通信指挥官（8 场景，综合应用题，无唯一标准答案，2026-07-20 大改）

- **规则变化**：CRC 改为默认开启、学生不可关，界面显示"CRC：基础检错，默认开启"。学生只选 5 项：① 调制档位（高速档/稳定档） ② FEC 强度（关闭/中等保护/强保护） ③ 重传策略（不允许/仅关键数据允许/全部允许） ④ 业务降级（全部保留/降低语音码率/降低图片分辨率/暂停图片/只保留位置和文字） ⑤ 业务发送顺序（位置坐标/求救文字/短语音/低清图片 的发送先后）。
- **场景组成**：链路（良好/一般/较差/非常差）× 剩余电量（充足/50%/30%/只够一小时）× 当前任务（报告坐标/发送求救文字/保持短语音联系/发送现场图片/综合救援通信）× 网络压力（较低/一般/拥塞）。
- **20 分 5 维评分**：核心任务保障 0-5（最重要数据是否优先传送）/ 可靠性 0-5（调制档位、FEC、重传是否符合链路条件）/ 资源开销把控 0-4（是否过度使用强FEC和全部重传）/ 电量管理 0-3（策略是否符合剩余电量）/ 业务降级 0-3（是否合理减少低优先级业务）。**评分明确不奖励过度保护**：强 FEC 只在链路差时才拿满可靠性分，链路好时反而扣资源开销分；全部重传在任何场景资源开销都要扣分，电量紧张时电量管理也扣分；业务降级维度按"链路+电量+网络压力"的综合严重度评估匹配度，不是降得越狠越好。
- **CRC 默认开启原因**：CRC-4 只需要极少开销就能提供最基础的检错能力，是通信可靠性的底线保障，因此本课把它设为默认开启且学生不可关闭——现实中"完全不做任何检错"不是常见的工程取舍点，这不是一个值得让学生练习"关掉"的选项。
- **FEC 强度怎样选择**：链路越差，越需要更强的 FEC 提前修复错误；但链路好时选强 FEC 不会带来更高的可靠性评价，反而会被资源开销把控维度扣分——FEC 强度要跟链路条件匹配，不是越强越好。
- **重传的收益与代价**：重传能在检测到错误后重新发一次换取正确性，但会增加流量、延迟和耗电；"仅关键数据允许重传"通常是性价比最高的选择，"全部允许"在任何场景都会被算作较高的资源开销，电量紧张时还会进一步拉低电量管理得分。
- **业务降级不等于允许 bit 错误**：业务降级是主动减少需要传输的数据（比如降低语音码率、暂停图片），不等于允许剩下的数据随便出错——降级之后仍要发送的数据依然需要 CRC 检错；低清图片即使降低分辨率或暂停，剩下发送的数据依然需要 CRC 检错，不存在"允许随意出错"的豁免。
- **三级提示**（本关固定算法，8 场景共用）：一级——先判断这次任务哪种数据最重要、最不能出错。 二级——再看当前链路和电量能不能承担强保护——强 FEC、全部重传都有资源开销和电量代价，不是选得越强越好。 三级——优先保住位置坐标和求救文字，再决定短语音、低清图片怎样降级、按什么顺序发送。

## 四、快速答案卡（可打印，供课堂随时翻查）

- **QPSK 映射**：右上 00，左上 01，左下 11，右下 10。
- **BPSK 映射**：左边的点 = 0，右边的点 = 1。
- **BER 公式**：BER = 错误 bit 数 ÷ 总 bit 数（信道 BER 分母是编码后总 bit 数；Payload BER 分母是原始 payload bit 数，两者必须分开标注）；第③关允许分数集固定为 1/8、2/8、3/12、2/16、4/16。
- **CRC**：CRC-4（生成多项式 x⁴+x+1，二进制 10011）——只检测错误，不自动修复数据；第④关默认开启，学生不可关闭。
- **FEC**：利用冗余修复一定能力范围内的错误，链路越差越需要更强的 FEC，链路好时选强 FEC 不加分反而扣资源开销分。
- **重传**：仅关键数据允许重传通常是性价比最高的选择；全部允许在任何场景都算较高资源开销，电量紧张时还扣电量管理分。
- **业务降级**：主动减少需要传输的数据，不等于允许数据随便出错，降级后仍要发送的数据仍需 CRC 检错。
- **海岛原则**：先保位置和文字，再考虑语音和图片；没有唯一正确组合，能说明取舍理由就是好方案。
- **两条写死边界**：① 模拟噪声强度和 bit 独立翻转概率是两个完全不同的参数，互不影响；② 误码率必须分别标注"信道 BER"和"Payload BER"，不能只写"BER"。
- **快捷键**：T 打开教师辅助（先弹隐私确认）／Esc 立即关闭／H 显示下一级提示／R 直接显示正确答案或推荐方案／N 下一题。

## 五、技术故障备用方案（网页不能用时的纸笔替代）

- **符号配对纸笔题**：老师口头报出一个 bit 串或象限方位，学生纸上写出对应的另一半（照第四节固定映射表判断），同桌互相检查。
- **判决区域纸笔题**：老师在黑板画一个简化的星座图坐标轴，点上一个"接收点"位置，学生纸上判断它离哪个理想点最近、是否跨过了判决边界。
- **BER 手算题**：老师口头给出两串等长 0/1（比如 8 位），学生纸上逐位比较，数出错误位置和错误数，再算出 BER（用第三关允许的简单分数形式）。
- **重复码多数表决题**：老师给出一个 bit 的三次重复副本（比如 1 1 0），学生纸上做多数表决，判断恢复出的 bit 是多少，并说明这是否是"多数表决失败"的情形（两份或以上不一致时）。
- **海岛策略卡**：把第三节"海岛场景推荐策略表"打印发给两组，直接讨论当前假设场景该怎么选，不需要网页也能完成第④关的教学目标。
